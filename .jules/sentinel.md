## 2024-05-31 - [Defense in Depth] Added Content Security Policy (CSP)
**Vulnerability:** The application was missing Content Security Policy (CSP) headers. Without a CSP, the application is more susceptible to Cross-Site Scripting (XSS) attacks, as any injected script would be allowed to execute.
**Learning:** The application requires specific allowances to function correctly. `viewer.html` uses an inline script tag for its import map which resolves `three` and its addons. The application also relies heavily on inline styles for dynamic UI updates across its JS files.
**Prevention:** Added `<meta http-equiv="Content-Security-Policy">` tags to `index.html`, `viewer.html`, and `reaction-lab.html`. The base policy is `default-src 'self'`. To support dynamic UI changes, `style-src 'self' 'unsafe-inline'` was allowed. For `viewer.html`, `script-src` was configured to allow `'self'`, the external CDN `https://cdn.jsdelivr.net`, and the exact SHA-256 hash of the inline import map script block to avoid enabling `'unsafe-inline'` for scripts, following the principle of least privilege.
